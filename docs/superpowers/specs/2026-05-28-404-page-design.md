# Page 404 — Design Spec

**Date:** 2026-05-28  
**Statut:** Validé

## Objectif

Remplacer la page 404 par défaut de Next.js par une page stylée, cohérente avec l'identité visuelle de DBS Store.

## Décisions design

| Décision | Choix |
|---|---|
| Style | Illustré (fond gris clair, icône centrée) |
| Navigation | Page autonome — pas d'AppBar |
| Langue | Français |

## Fichier à créer

`app/not-found.tsx` — racine du projet (level global, bypasse tous les layouts).

## Structure visuelle

Page centrée verticalement et horizontalement (`min-h-screen`, flexbox).  
Fond : `bg-muted/50`.

De haut en bas :
1. **Logo** — `Image` Next.js pointant vers `/images/dbs-store-logo.png`, hauteur 36px
2. **Icône** — cercle blanc avec ombre douce, icône `PackageX` de lucide-react (boîte avec croix)
3. **Titre** — `"Page introuvable"`, `text-2xl font-bold`
4. **Sous-titre** — `"Ce lien ne mène nulle part ou la page a été déplacée."`, `text-muted-foreground`
5. **Bouton** — composant `Button` Shadcn (`asChild` + `Link` Next.js vers `/`), label "Retour à l'accueil" avec icône `Home`
6. **Code d'erreur** — `"ERREUR 404"` en `text-xs text-muted-foreground/50` tout en bas

## Contraintes techniques

- Fichier **client component non requis** — pas d'état, `not-found.tsx` peut être un Server Component
- Pas de `export const dynamic` nécessaire — aucun accès DB ni binding Cloudflare
- Utiliser `Button` de `@/components/ui/button` et `Image` de `next/image`
- Icône `PackageX` depuis `lucide-react`
- Métadonnées : `export const metadata = { title: "Page introuvable — DBS Store" }`

## Ce qui n'est pas dans le scope

- Barre de recherche
- Suggestions de produits
- Liens vers les catégories
